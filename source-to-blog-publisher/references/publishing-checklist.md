# Publishing Checklist

Use this checklist after drafting and before pushing.

## Source

- Source URL recorded
- Source author/account recorded if available
- Source title recorded if available
- Any source images copied into the post asset folder if used

## Writing

- The post is original prose, not close paraphrase
- Any expansion beyond the source is clearly framed as interpretation or extension
- The tone matches the target repo
- The opening paragraph tells readers what the note is about without sounding generic

## Repo Fit

- Frontmatter matches existing post conventions
- Permalink pattern matches the repo
- Asset paths render correctly
- Index or notes landing page updated if needed

## Git Hygiene

- `git status` checked before staging
- Only article-related files staged
- Commit message describes the post clearly
- Push only after confirming branch and remote
